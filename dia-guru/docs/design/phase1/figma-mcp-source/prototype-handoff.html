08 Prototypes + 09 Handoff Specs (v2)

Clickable flows are now centered on decisions and outcomes: capture, conflict handling, check-in completion, and quick-entry from widgets/live activity.

Prototype Flow Map

Primary click-through sequence for review and sign-off.

1. First run -> Auth -> Home
Sign-in success lands on Home hero + quick capture composer.
2. Home quick capture -> auto-schedule success
Includes queued/scheduled preview and refresh state.
3. Home capture -> DeepSeek follow-up -> scheduled
No duration input path, one clarifying question, then schedule.
4. Conflict sheet decisions
Overlap, make-room rebalance, auto-decide, cancel.
5. Overdue check-in complete/reschedule
Includes "Why this time?" explanation target.
6. Widget tap -> quick capture sheet
Source-tagged as widget and keyboard-focused on open.
7. Live Activity tap -> check-in target
Action routing to complete or reschedule path.
